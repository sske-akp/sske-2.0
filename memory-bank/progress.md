# progress.md

This document tracks what works, what's left to build, current status, known issues, and the evolution of project decisions.

**What Works:**

- Interactive, editable tables for sales and purchase invoices
- Modular UI components (combobox, checkbox, input)
- Product and invoice data models
- Responsive, modern UI with Tailwind and Radix UI

**What's Left to Build:**

- Persistent storage for invoices, products, and inventory
- Customer and supplier management features
- Dashboard and reporting modules
- Advanced validation and error handling

**Current Status:**

- Core invoice editing and product selection features are functional
- UI and component architecture is modular and extensible

**Known Issues:**

- No backend or persistent storage implemented yet
- Limited validation and error feedback
- Dashboard and analytics features not started

**Project Evolution:**

- Initial focus on modular, reusable UI and data models
- Iterative addition of business features and modules
